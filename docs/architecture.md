## 五层架构

```
┌──────────────────────────────────────────────────────────┐
│                    用户交互层                              │
│        自然语言输入  ·  进度反馈  ·  人工确认 UI           │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│              决策规划层（本地 VLM）                        │
│    ┌──────────────────────────────────────────┐          │
│    │  Qwen2-VL-7B（本地推理，INT4 量化）       │          │
│    │  职责：意图解析 + 任务分解 + 动作决策     │          │
│    │  显存：~9GB（含 ViT）                     │          │
│    └──────────────────────────────────────────┘          │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│              视觉感知层（本地 CV 模型）                    │
│    ┌────────────────┐  ┌──────────────┐  ┌────────────┐ │
│    │ YOLOv8m        │  │ PaddleOCR    │  │ SoM 标注   │ │
│    │ UI 元素检测    │  │ 中文文字识别 │  │ 编号框生成 │ │
│    │ 显存 ~2GB      │  │ 显存 ~1GB    │  │            │ │
│    └────────────────┘  └──────────────┘  └────────────┘ │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│              安全控制层                                   │
│    风险评级  ·  人工确认门禁  ·  审计日志  ·  预算熔断    │
└────────────────────────────┬─────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────┐
│              执行层                                       │
│    PyAutoGUI（鼠标键盘）· Pywinauto（Win 控件）          │
│    Playwright（浏览器）· 文件系统 API                    │
└──────────────────────────────────────────────────────────┘
```

## OODA 运行时序

```
用户输入指令
    ↓
[决策层] 解析意图 → 分解为 N 步任务
    ↓
[循环开始]
    ↓
[视觉层] 截图 → YOLO 检测 → OCR → SoM 标注 → 生成带编号的图
    ↓
[决策层] Qwen2-VL 看图 + 当前任务步骤 → 输出动作（如 click 元素2）
    ↓
[安全层] 校验：动作风险等级？是否需要人工确认？
    ↓
[执行层] 执行动作（PyAutoGUI / Pywinauto / Playwright）
    ↓
[视觉层] 再次截图 → 检查任务是否完成
    ↓
[循环结束] → 任务完成 → 通知用户 + 写审计日志
```

## 显存分配

| 模块 | 模型 | 显存 | 策略 |
|------|------|------|------|
| 主决策 | Qwen2-VL-7B INT4 | ~5GB | 常驻 |
| 视觉编码器 | Qwen2-VL ViT | ~3GB | 常驻（与主模型共享） |
| UI 检测 | YOLOv8m | ~2GB | 常驻 |
| OCR | PaddleOCR | ~1GB | 常驻 |
| KV Cache | 推理缓存 | ~2GB | 动态 |
| 图像缓冲 | 截图/标注图 | ~1GB | 动态回收 |
| **合计** | | **~14GB** | **留 2GB 余量** |

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 视觉驱动 vs 控件树 | 视觉驱动为主，控件树为辅 | 通用性强，能操作任意 GUI |
| 云端 vs 本地 | 全本地 | 隐私、成本、可定制 |
| FP16 vs INT4 | INT4 量化 | 显存预算决定 |
| 一次一个 vs 批量 | 一次一个任务 | Harness Engineering 规范 |
