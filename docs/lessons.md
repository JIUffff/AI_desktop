# 失败复盘积累

> 本文件记录每次失败的原因和教训。每次会话开始时读它，避免重犯同样的错误。
> 这是 Harness Engineering 的"免疫系统"——文档随错误进化，AI 越来越少犯错。

---

## 记录格式

```markdown
## [YYYY-MM-DD] T-XX 任务简述
- **现象**：发生了什么
- **原因**：为什么发生（归因到需求/技术/规则）
- **修复**：怎么修的（代码改动）
- **新规则**：在哪个文档增加了什么规则，防止再犯
```

## 常见错误模式（积累后提炼）

随着项目推进，此处会提炼出高频错误模式。例如：

- 显存相关：加载模型前未检查剩余显存
- 坐标相关：高 DPI 下截图坐标与鼠标坐标不一致
- 依赖相关：任务依赖未完成就提前开始
- 文档相关：改了代码但没更新 Spec
