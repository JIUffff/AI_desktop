核心结论

UI-TARS-1.5-7B 是 2026 年 GUI Agent 领域的 SOTA 开源模型。它在 ScreenSpot-V2（94.2%）、ScreenSpotPro（61.6%）、OSWorld（42.5%）三项核心基准上全面超越 OpenAI CUA 和 Claude 3.7，且专为「截图→推理→动作」流程设计，完美匹配本项目 OODA 循环架构。AWQ INT4 量化后仅需 ~6GB 显存，在 RTX 5070 Ti 16GB 上运行从容。

选型理由详解

为什么选 UI-TARS-1.5-7B 作为主模型？

理由详细说明
1. GUI 任务 SOTA ScreenSpot-V2 94.2%（超 OpenAI CUA 6.3%）、ScreenSpotPro 61.6%（超前任 SOTA 18%）、OSWorld 42.5%（超 OpenAI CUA 6.1%）。在 PC 控制场景无可匹敌。
2. 原生 GUI Agent 架构 专为「截图→推理→动作」设计，输出 thought + action，完美匹配 OODA 循环。无需像通用 VLM 那样额外封装 prompt。
3. 推理能力 支持 RL 训练的推理能力，先思考再操作，复杂任务成功率更高。
4. 显存可控 AWQ INT4 量化后 ~6GB，加 ViT 共 ~8GB，在 16GB VRAM 中留有充足余量给 YOLO/OCR/Cache。
5. Apache 2.0 商用许可 允许商业使用，无法律风险。
6. 字节跳动持续维护 UI-TARS-1.5 是 2025 年最新版，配套 UI-TARS-desktop 应用和 SDK，生态完整。

为什么保留 Qwen3-VL-8B 作为备选？

理由详细说明
1. 极致省显存 GPTQ-Int4 量化后仅 ~3.1GB，为 16GB 显存中最省的 8B 级 VLM，留 4GB 余量。
2. 通用 VLM 能力强 当需要非 GUI 视觉任务（文档理解、图片问答、复杂 OCR）时，Qwen3-VL 更通用。
3. 2025.10 最新发布 Interleaved-MRoPE + DeepStack 架构创新，256K 上下文，32 语言 OCR。
4. Instruct + Thinking 双模式 Instruct 快速响应，Thinking 深度推理，按场景切换。

风险与缓解措施

风险等级缓解措施
UI-TARS-1.5-7B 在特定 Windows 应用上表现未验证 中 准备 20-30 张标准测试截图，在 T-19（端到端测试）中覆盖典型场景；若成功率 <65%，切换到 Qwen3-VL-8B 对比
AWQ 量化导致精度损失 中 量化后跑 ScreenSpot 子集基准，精度损失 >5% 则回退到 INT8 或使用 FP8 版本
vLLM 0.5.0+ 坐标解析异常 高 锁定 vLLM 0.4.2 版本；使用 smart_resize 校准坐标转换；CI 中加入坐标范围 [0,1000] 验证
UI-TARS 坐标输出格式与 SoM 编号不兼容 中 在 vlm_server.py 中做格式适配层，统一输出 {action, coordinates, element_id}
模型下载失败（网络问题） 中 强制代理配置（AGENTS.md §13.2）；支持 HF 镜像站；断点续传
UI-TARS 中文场景覆盖不足 低 UI-TARS 训练数据含中文 GUI；备选 Qwen3-VL 中文能力更强，可随时切换
